# Memory bank (agent workflow)

Read order for a new session:

1. `projectBrief.md` — scope, success criteria, priorities.
2. `activeContext.md` — what matters right now; update when phases shift.
3. `progress.md` — checkbox reality; flip items when work lands.
4. `migration-inventory.md` — legacy feature/API surface from Go + Vuetify.
5. `architecture.md` — target layering and what not to copy from Go.
6. `techDecisions.md` — Nostrum, Horde, Redis, DB split, STT/TTS notes.

Maintenance: after meaningful implementation batches, edit `activeContext.md` and `progress.md`. Keep `migration-inventory.md` accurate if legacy parity scope changes.
